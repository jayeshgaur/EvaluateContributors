# How We Work

Ship working code fast. Get something visible and running early, then iterate.
Working > perfect. Simple > clever. Less code > more code.

# Workflow

- Understand the problem before writing code — ask clarifying questions if needed
- Start with the simplest approach that could work
- Run the app/server early so we can see results in real time
- When debugging, read the error and fix root cause — no workarounds or suppressions
- Install missing dependencies without asking
- Keep responses short — no summaries or explanations unless asked

# Code

- Default to Python unless the task clearly calls for something else
- Prefer popular, well-supported tools over obscure ones
- Minimal dependencies — don't install things we might not need
- No boilerplate comments, no unnecessary abstractions
- If there's a choice, prefer convention over configuration

# Testing

- Run single tests, not full suites, for speed
- Test behavior, not implementation

# When Compacting

Preserve: modified files list, current task/goal, key decisions, any running commands or servers.
